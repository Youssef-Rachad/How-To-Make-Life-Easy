Onto the Coding!

What is HTML?

HTML stands for Hyper Text Markup Language. Think of it an author, it write all of the content of your webpage, makes sense of titles, headings, text and images. Formely, HTML is a scripting language.

What is CSS?

Adding styling is done through the Cascading Style Sheets language CSS. Think of this guy as the illustrator of the book. Working side by side with our author, CSS enhances the quality of our website, taking it from beginner using bad software *ahem Notepad* to amateur making a blog to get organized and from amateur to Silicon Valley professional.

How does our artist communicate with our author, you may ask? Easy! We set up their communication through the <link> tag in the header with a path to the .css file <link rel=“style” > To keep things simple and organized, I suggest you keep the .html and .css files next to each other. As you get comfortable working with files, you will be able to link files from anywhere. Lastly, I would like to introduce the Box Model which defines the spacing of an element.

Briefly said, the border is a line that surrounds the content. You may want to space it away from your content; thus the padding comes into play. When spacing out elements between each other, a margin is specified. Both the padding and the margin are spaces that aerate your content.

What is JavaScript?

Last but most certainly not least is JavaScript, or JS for short. This is a programming language used to jazz up your website. Need show/hide content? Worried about getting the days right in your calendar? Worried that your to-do list shows up correctly? JavaScript has you covered! Js say the word. Useful resources when learning

All of the web development languages have been developped, however not documented in a traditional way. Instead of picking up a book or taking a class, the twenty first century instructs us to use the web. In fact, here is a list of the leading sources when it comes to learning HTML, CSS and JavaScript.

  • W3 Schools has information about every obscure corner of web development. I use this as my primary look up source when I need to know how a CSS stylesheet is linked, for example.
  • Mozilla Developer Network, or MDN for short. The makers of the FireFox browser themsleves document all there is to know about HTML. Their documentation might be long to read, but it sure is complete.
  • Stack Overflow. No list of coding resource would complete without these guys. If you have a problem, chances are someone else had it too. The solution to that problem is probably there, waiting for you. In just a few clicks, you can also ask your own questions. Just a tip: their website is slightly harder to navigate. I prefer to google the problem and add 'Stack' at the end to find the right page.
